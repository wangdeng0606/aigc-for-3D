# BlockGen AI

BlockGen AI 是一个以“搭积木”思维为核心的三维可控式 AIGC 工具。  
它旨在解决当前文生视频 / 图生视频 / AIGC 影视生成过程中，**物体无法被精确摆放、镜头无法被稳定控制、空间关系难以约束**的问题，让设计师、导演与创作者能够在三维空间中像“搭场景”一样构建画面，从而提升 AIGC 视频生成的可控性、可预期性与生产效率。

---

## 项目背景

当前主流 AIGC 视频工具虽然在画面风格、镜头质感与生成效率上进步迅速，但在影视生产和三维创作场景中，仍然存在几个核心问题：

- 无法精确控制物体在三维空间中的位置、朝向和层级关系
- 镜头运动轨迹难以稳定复现
- 场景中多个元素之间的空间逻辑容易失真
- 复杂镜头语言难以通过纯文本准确表达
- 用户“知道自己想要什么”，但很难通过提示词完整描述

因此，AIGC-for-3D 提出一种新的工作方式：

> 先在三维空间中完成“场景搭建、物体摆放、镜头设计与运动录制”，再将结构化场景信息与生成意图一起传递给 AIGC 视频模型，从而实现更高可控度的视频生成。

---

## 核心理念

### 1. 像搭积木一样做 AIGC
用户不再只依赖文本描述，而是可以直接在三维空间中拖拽、摆放、旋转和缩放对象，像搭建白模场景一样快速完成画面构图。

### 2. 先搭场，再生成
AIGC-for-3D 将“空间控制”前置，把传统 AIGC 中最不稳定的部分——位置关系、镜头轨迹、物体布局——变成可编辑、可记录、可复用的三维过程。

### 3. 让提示词从“描述”走向“调度”
文本不再承担全部控制功能，而是与三维布局、镜头参数、动画路径共同构成一个完整的生成输入系统。

## 核心功能

### 场景搭建
- 提供基础三维模型库，用于快速搭建白模场景
- 支持物体拖拽、平移、旋转、缩放
- 支持多物体组合与层级管理
- 支持场景预设与模板复用

### 镜头控制
- 支持虚拟摄影机创建与管理
- 支持焦段、景别、滤镜等镜头参数设置
- 支持关键帧方式定义镜头运动
- 支持手动录制镜头移动轨迹

### 动态录制
- 支持对可动对象进行路径录制
- 支持用户在录制过程中同步移动镜头与物体
- 支持生成结构化运动数据，供后续 AIGC 调用

### AIGC 生成接口
- 将场景信息、镜头信息、运动轨迹与文本提示词打包输出
- 对接视频生成 API
- 支持基于已有三维布局进行可控式视频生成
- 预留多模型兼容能力，适配不同 AIGC 服务商

### 可视化创作流程
- 用三维可视化替代纯提示词试错
- 降低导演、设计师、内容策划等非技术用户的使用门槛
- 强化创意到生成结果之间的可解释性与一致性

## 典型使用流程

1. **选择基础场景或新建项目**  
   用户进入系统后，可从空白场景开始，也可加载模板场景。

2. **搭建三维布局**  
   从素材库中拖入角色、道具、建筑、背景等对象，并完成空间摆放。

3. **设置镜头语言**  
   创建摄影机，调整焦距、机位、朝向与滤镜参数。

4. **录制镜头与物体运动**  
   用户可像操作虚拟摄影机一样录制镜头运动，同时对场景中可动对象进行路径录制。

5. **输入生成意图**  
   补充风格、氛围、材质、光影、叙事等文本提示词。

6. **提交至 AIGC 视频模型**  
   系统将三维布局数据、镜头数据、运动轨迹与文本提示词组合后，调用生成接口输出结果。

7. **迭代与优化**  
   用户可基于已有项目反复微调位置、镜头和提示词，以获得更稳定的生成结果。

## 解决的问题

AIGC-for-3D 重点解决以下问题：

| 问题 | 传统 AIGC 痛点 | AIGC-for-3D 的解决方式 |
|------|----------------|-------------------------|
| 物体位置不可控 | 提示词难以精确描述空间关系 | 通过三维拖拽与摆放直接定义位置 |
| 镜头难控制 | 运镜语言不易稳定生成 | 通过虚拟摄影机和轨迹录制控制镜头 |
| 多元素关系混乱 | 多个主体相对位置经常失真 | 通过空间布局建立稳定的结构约束 |
| 反复试错成本高 | 纯文本生成依赖大量提示词调参 | 用可视化搭建替代部分文本试错 |
| 创作者表达门槛高 | 非技术用户难写复杂提示词 | 用“搭场景”方式降低表达成本 |

## 产品优势

### 更高的可控性
将“生成前控制”引入 AIGC 流程，使空间关系与镜头语言不再完全依赖模型随机性。

### 更适合影视制作逻辑
相比纯文本生成，该工具更接近导演、摄影、美术和分镜师的工作方式。

### 更低的沟通成本
团队成员可以直接基于三维场景讨论构图、机位、走位与运动，不必反复解释抽象提示词。

### 更强的扩展能力
工具既可以作为独立的 AIGC 前端控制器，也可以作为后续导演系统、智能拍摄系统、数字剧组系统的基础模块。

## 技术思路

AIGC-for-3D 的整体架构可分为四层：

### 1. 交互层
负责用户在前端进行三维搭建、镜头控制与录制操作。

### 2. 场景数据层
负责保存：
- 物体位置、旋转、缩放
- 摄影机参数
- 动画轨迹
- 场景层级关系
- 灯光与基础环境参数

### 3. 生成编排层
将三维结构化信息转换为适合 AIGC 模型理解的控制输入，包括：
- 场景语义摘要
- 镜头运动描述
- 对象布局约束
- 文本提示词组合

### 4. 模型接口层
连接外部视频生成 API，实现最终画面生成。

## 项目愿景

AIGC-for-3D 希望成为连接“三维创作逻辑”与“AIGC 生成能力”的中间层工具。  
它并不是要替代生成模型，而是为生成模型提供一个更清晰、更稳定、更符合创作者思维方式的输入界面。

我们相信，未来的 AIGC 创作不应只是“写提示词”，  
而应该是：

> **在可视化空间中组织世界，再让 AI 按照你的结构去生成。**
